Chassis Policies
Chassis Policies

Chassis policies in Cisco Intersight allow to configure various parameters of the chassis, including IP pool configuration, VLAN settings, SNMP authentication, and SNMP trap settings. A chassis policy can be assigned to any number of chassis profiles to provide a configuration baseline for a chassis.

To launch the Policies Table View, choose Configure > Policies.

The Chassis Policy creation wizard in Cisco Intersight has two pages:

  • General—The general page allows to select the organization and enter a name for policy. Optionally, include a short description and tag information to help identify the policy. Tags must be in the key:value format. For example, Org:IT or Site: APJ

  • Policy Details—The policy details page has properties that are applicable to UCS Chassis Policies.

Chassis Policies can also be cloned by using the Policy Clone wizard with properties that are similar to the existing policies. The clone policy action is available on both the policies list and detailed views. For more information, see Cloning a Policy.

You can compare up to five policies of the same type from the Policies List View page to identify configuration differences, maintain consistency, and simplify troubleshooting.

To compare policies, select the checkboxes next to the desired policies, click the ellipsis (…) at the top left of the page, and choose Compare. On the Compare side drawer, only the differences between the selected policies are shown by default. To display all parameters, select All from the Display drop-down menu.

Note:

You can compare a minimum of two and a maximum of five policies of the same type at a time.

The following list describes the chassis policies that you can configure in Cisco Intersight.

  • IMC Access Policy—This policy enables configuration and management of the network by mapping IP pools to the chassis profile. It allows In-Band and Out-of-Band configurations, with association to an IP address from the IP pool. The policy also permits forwarding SNMP traffic over In-Band and Out-of-Band configurations.

  • Power Policy—Enables the management of power usage for the chassis. This policy allows to configure the redundancy mode of the Chassis Power Supply Units (PSUs) and allocate power to the chassis. You can view the redundancy health, redundancy mode, input power health, and output power health of the chassis in the properties section of the General tab on the Chassis details view page. For Cisco UCS X9508 Chassis, you can configure Power Save Mode and Dynamic Power Reallocation.

  • SNMP Policy—Configures the SNMP settings for sending fault and alert information by SNMP traps from the managed devices. SNMP Users or SNMP Traps configured previously on the managed devices are removed and replaced with users or traps that you configure in this policy. If you have not added any users or traps in the policy, the existing users or traps on the input/output module (IOM) are removed.

  • Thermal Policy—Allows to set the value of the Fan Control Mode for the chassis. The Fan Control Mode controls the speed of the chassis fan to maintain optimal server cooling.

Chassis Profiles
Chassis Profiles

A UCS Chassis profile enables you to create and associate chassis policies to an Intersight Managed Mode (IMM) claimed chassis. When a chassis profile is associated with a chassis, Cisco Intersight automatically configures the chassis to match the configurations specified in the policies of the chassis profile. The chassis-related policies can be attached to the profile either at the time of creation or later.

Important:

  • The chassis profile feature is available in Intersight only if:

    • You have installed the Intersight Essentials License.

    • You are either an Account Administrator or Server Administrator.

  • Policies that are attached to a chassis profile can be created ahead of creating a profile or during the creation of the profile.

  • If chassis polices are changed after deployment, the chassis profile will be set in Pending Changes state and you must manually re-associate the changed policies to the chassis.

  • Chassis policies will be applied to both the input/output modules (IOMs) in a chassis. The chassis policies association workflow will fail even if the policy cannot be applied to any one of the IOMs.

    Note:

    The IMC Access Policy and SNMP Policy are currently not supported for Cisco UCS X-Series Direct chassis (UCSX-9508).

Creating a UCS Chassis Profile

A Chassis Profile configures a chassis using reusable policies.

  1. Log into Cisco Intersight with your Cisco ID and select admin role.

  2. Navigate to Configure > Profiles > Chassis Profiles tab and click Create UCS Chassis Profile.

  3. On the General page, do the following:

    1. Organization—Select the organization from the list. This field parameter supports the capability of configuration sharing across Organizations.

    2. Name—Enter a name for your profile.

    3. (Optional)Description and Tag—Enter the description and tag to identify the profile.

    4. Click Next.

  4. On the Chassis Assignment page, assign a chassis to the Chassis Profile. You can also click Assign Later and assign a chassis to the chassis profile later.

  5. Click Next.

  6. On the Chassis Configuration page, attach the required policies and click Next.

  7. On the Summary page, verify the details of the UCS Chassis Profile and the policies attached to it.

  8. Click Deploy to deploy the UCS Chassis Profile to the assigned Fabric Interconnect.

Note:

  • For more information, see Configuration Sharing Across Organizations.

  • For more information on cloning a profile using configuration sharing across Organization, see Cloning a Profile Across Organization.

UCS Chassis Profile Details

On the UCS Chassis Profile Details page, you can:

  • Perform chassis profile Actions:

    • Deploy—Deploy the chassis profile on a Fabric Interconnect pair.

    • Edit—Edit the properties of the chassis profile.

    • Unassign Chassis—Unassign the chassis profile from the Fabric Interconnect pair.

    • Set User Label—Allows you to set, update, or delete user labels for each chassis profile. It must be between 1 and 64 alphanumeric characters, containing only the following special characters: ! # $ % & * + , ( ) [ ] { } | / . ? @ _ : ; ~

  • View the UCS Chassis profile Details:

    • Status—The status of Chassis profile deployment on a Fabric Interconnect pair. The status can be:

      • OK

      • Not Assigned

      • Not Deployed

      • Failed

      • Not Deployed Changes

    • Name—The chassis profile name.

    • Chassis—The chassis details.

    • Last Update—The date and time when the chassis profile was last updated.

    • Description—The description of the chassis profile.

    • User Label—Displays the assigned user label that helps in identification of the chassis profile.

    • Organization—The selected organization is displayed. Click default to set a default organization.

    • Tags—The existing tags for the selected object are displayed, by default. Click Set to add new tags or modify the existing ones.

  • View the Policies that are attached to the chassis profile.

Note:

Some of the columns are disabled by default, such as, User Label. To view such columns in the chassis profile table view, you need to enable them while customizing the table view.

Chassis Profile Templates
Chassis Profile Templates

Chassis profile templates enable you to define a template from which chassis profiles can be easily derived and deployed at scale. This method is particularly useful when you need a large number of chassis profiles that use the same set of policies. Any property modification made in the template is inherited by the derived profiles. You can deploy these modified profiles individually.

Chassis Profile Template List View

When you select Infrastructure Service > Templates in the Intersight UI, the Chassis Profile Templates List View is displayed.

The list view shows the following details in tabular format:

  • Description—The description of the template provided during template creation.

  • Last Update — The date on which the template was last updated.

  • Name—The name of the chassis profile template.

  • Organization—The name of the organization.

  • Usage—The number of chassis profiles derived from the template.

You can create up to 80 chassis profile templates at a time. For more information, see Creating Chassis Profile Template and Deriving Profiles.

Chassis Profile Template Actions

After creating chassis profile template, actions that can be performed on the chassis profile template are as follows:

  • Derive Profiles– You can derive multiple profiles from a single template. You can use the option of Derive Profiles while creating a template or use the option of Derive Profiles present in the Actions menu in the template list view.

    Each template supports creating a maximum of 80 profiles at one time.

  • Edit–You can edit the template at any time. Any edit made in the template gets reflected in the derived chassis profiles. The status of all the derived profiles changes to Not Deployed Changes after the template update.

    The Not Deployed Changes status will be shown only for deployed profiles whose status is OK, else it will not be shown.

  • Clone– You can clone the chassis profile template into many copies. Cloning allows you to reuse the same configuration for multiple templates. Each copy can have a distinct description, name, prefix, and suffix.

  • Delete–You can delete a chassis profile template only when there are no derived chassis profiles attached to that template. The Usage value must be 0.

  • Attach to Template–You can attach the chassis profile to any available template. All the policies in the template will be applied to the profile.

  • Create a Template–A chassis profile can be used to create a template. This template can then be used to create multiple profiles with the same configurations and deployed on multiple chassis.

    The profile configuration will be applied to the template. While template creation, if you toggle ON the Attach UCS Chassis Profile to Profile Template button, the selected profile gets attached to the template under creation. If you keep the toggle button OFF, the selected profile's properties are carried to the template but the profile is not attached to it.

  • Detach from Template – You can detach the profile from a template.

    Note:

    • Create a Template and Attach to Template actions can be performed only if a profile is not attached to any template.

    • A profile can be attached to an existing template. This attachment overrides the configurations of the profile and replaces them with the template properties.

    • A profile attached to a template cannot be modified. The modifications can be done in the associated template.

    • A profile can be detached from a template, and then modified as necessary.

    • A detached server profile can always be reattached to a template.

Chassis Profile Template Detail View

You can click on the name of the template to go to the Chassis Profile Template Detail View.

The detail view shows the configuration properties of the template and the details of the attached profiles. It also displays an Action button that lists the actions that can be performed on the template.
