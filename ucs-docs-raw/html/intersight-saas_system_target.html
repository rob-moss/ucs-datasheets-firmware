Target Management
Target Management

To get started with Cisco Intersight, you must claim a target for use with the Cisco Intersight platform. In Intersight, you will need the device ID and claim code, both of which are unique to the target and retrieved from the target. For Intersight Virtual Appliance, you will need to enter the IP/Hostname of the target you want to claim, the Username for the device, and the Password for the user. This user must have administrative privileges.

You can also unclaim targets or remove them from your list, and, reclaim later as required. For more information, go to Unclaim Targets.

Targets Details View
Targets Details View

When you select a target in the Targets table view, a Targets detail page with information specific to the claimed target is displayed. You can view the following information in the Targets details view:

  • Name

  • Type—Standalone Server, Cisco UCS Domain, Cisco UCS Director, Cisco HyperFlex Cluster, or VMware vCenter

  • FI-A/FI-B/Server/HyperFlex Cluster—From here, you can navigate to the Details page for the corresponding target (except Cisco UCS Director)

  • IP Address

  • External IP Address

  • Target ID—For more information about the format of the Target IDs, see Cisco Intersight Setup

  • Last Update—Timestamp of the target status refresh

  • Claimed Time—Date and time of the target claim

  • Claimed By—ID of the Intersight user

  • Connector Version—The latest Device Connector version

  • Product ID

  • HSM Enabled—Displays the status of the Hardware Support Manager (HSM) in VMware vCenter targets.

  • Intersight Assist—Displays the Intersight Assist selected during the target claim.

  • Organizations—Lists the Organizations to which the targets are assigned through Resource Groups.

  • Resource Groups—Lists the Resource Groups to which the targets are grouped.

Device Connection
Device Connection

Supported systems are connected to Cisco Intersight through a device connector that is embedded in the management controller of each system. The device connector provides a secure way for the connected targets to send information and receive control instructions from the Cisco Intersight portal, using a secure Internet connection.

Note: For more information on supported software versions, see Supported Systems.
